# 总需求
给我写一个python CLI 程序，实现一个本地文档片段语义查询,
允许使用第三方库，使用主流的技术，索引的构建有缓存机制，只有初次使用进行embedding，构建完毕就只增量更新
1. 不要使用三方的 向量数据库服务，使用本地运行的向量数据库
2. 默认使用开源的 `sentence-transformers` 小模型 `paraphrase-MiniLM-L3-v2`（CPU 友好），可通过环境变量 `LOCAL_EMBEDDING_MODEL` 替换为其他免费模型

# 项目目录要求如下：
data/
	1.txt
	2.txt
	....
tmp/
    xxx(存储向量数据)
src/
	xxx.py
	....
main.py


# 交互上：
按键1: 将数据库建立embedding
按键2： 查询文章(只返回文章的出处片段) 
按键3： 全部重新建立倒排索引
按键exit: 退出交互


# 数据
data下的数据为中英文混合的文章，字数大概在10W字以内

## 运行说明

### 目录结构
```

tmp/    # 索引缓存存放位置（）

```
